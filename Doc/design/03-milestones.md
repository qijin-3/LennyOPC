# 03 — 里程碑系统

## 3.1 设计理念

**里程碑是游戏进度的唯一驱动力。**

不再有「第几周」或「破冰期/起步期」这样的时间阶段。游戏进度由玩家自己设定和完成的里程碑来决定。

每个玩家的游戏路径都不同：
- 有人快速找到付费客户，7 天完成第一个里程碑
- 有人花更多时间学习和建立人脉，30 天才开始变现
- 这两种路径都是合法的，游戏不评判节奏

## 3.2 OPC 目标设定

在 Onboarding 阶段 4，玩家与 Lenny 对话，确定以下三要素：

```
OPC 方向：玩家的具体服务/产品定位
  例：「B2B SaaS 产品顾问」「个人品牌内容创作者」「AI 工具开发者」

目标受众：谁会为你的 OPC 付钱
  例：「10-50人初创公司的 CPO」「独立开发者」

宏观目标：玩家想在什么时间内达到什么状态
  例：「3个月内获得第一个付费客户」「6个月月收入¥10,000」
```

## 3.3 里程碑拆解

Lenny 将宏观目标拆解为 3-5 个具体里程碑，写入 TodoApp。

### 示例：B2B 产品顾问路线

```
宏观目标：6个月内月收入¥10,000

拆解里程碑：
1. [Day 7]  确定服务定价和方案（高优先级）
2. [Day 30] 找到第一个付费客户（高优先级）
3. [Day 60] 完成 3 次付费咨询（普通优先级）
4. [Day 90] 建立个人品牌（发布 3 篇文章）（普通优先级）
5. [Day 120] 月收入达到 ¥10,000（普通优先级）
```

### 示例：SaaS 产品路线

```
宏观目标：Product Hunt 发布并获得 200 赞

拆解里程碑：
1. [Day 14] 完成产品 MVP 设计稿
2. [Day 30] 获得 5 位早期测试用户
3. [Day 60] 收到第一笔付费（哪怕 ¥1）
4. [Day 90] Product Hunt 发布准备完成
5. [Day 100] Product Hunt 发布
```

## 3.4 里程碑数据结构

```js
// GameState.opc_goals
{
  direction: 'B2B SaaS 产品顾问',
  targetAudience: '10-50人规模的初创公司',
  goal: '6个月内月收入¥10,000',
  createdAt: 5,            // 游戏第几天设定
  milestones: [
    {
      id: 'ms_001',
      title: '确定服务定价和方案',
      priority: 'high',
      dueDay: 7,           // 游戏第几天截止
      completed: false,
      completedAt: null,
      source: 'lenny_onboarding'
    }
    // ...
  ]
}
```

## 3.5 里程碑完成触发

完成里程碑时（在 Todo 中勾选），触发对应叙事事件：

| 里程碑类型 | 触发效果 |
|-----------|---------|
| 第一个付费客户 | 王子墨发 Telegram 庆祝 + 解锁 Safari（若未解锁）|
| 完成首次付费咨询 | 信任度提升 + AI 事件引擎评分更新 |
| 内容发布（文章/视频）| 声誉 +5，可能触发媒体关注事件 |
| Product Hunt 发布 | 大幅声誉提升，触发社群事件 |
| 达到月收入目标 | 解锁成就 + 触发下一阶段叙事 |

## 3.6 里程碑调整

玩家可以在任何时候：
- 和 Lenny（或任何专家）重新讨论目标
- 手动在 Todo 中添加/修改/删除里程碑
- 请 Lenny 重新拆解（触发新一轮 AI 定向对话）

## 3.7 系统预置里程碑（Onboarding 完成前）

这些是游戏内置的、Onboarding 阶段强制推进的待办：

| 待办 | 来源 | 优先级 |
|------|------|-------|
| 思考接下来做什么 | 裁员邮件确认后 | 高 |
| 在 Safari 搜索 Lenny 并在 X 私信他 | 王子墨推荐后 | 高 |
| 与 Lenny 确定你的 OPC 方向 | 添加 Lenny 后 | 高 |

## 3.8 游戏结束条件

**唯一的失败条件**：现金归零。

- 触发「资金耗尽」叙事事件
- 玩家可选择「重新开始」（保留部分人脉记忆）或「结束」
- 没有「成功结局」的强制终点——玩家完成了自己的目标，就算完成了

## 3.9 进度显示

Menu Bar 显示：
- 游戏天数（「第 12 天」）
- 当前现金
- 时钟和速度控制

Numbers 看板显示：
- 当前现金和燃烧率
- 声誉、人脉
- OPC 方向（Onboarding 完成后）
- 里程碑完成进度（X / N 个）

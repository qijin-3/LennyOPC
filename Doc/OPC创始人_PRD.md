# OPC创始人 — 产品需求文档 (PRD)

**文档版本**：v2.0  
**最后更新**：2026-04-01  
**产品类型**：浏览器端叙事策略游戏（单文件，本地运行）  
**目标平台**：桌面端浏览器（优先），移动端自适应

## 一、产品概述

### 1.1 产品定位

《OPC创始人》是一款以「伪MacBook桌面」为核心界面的 AI 驱动叙事策略游戏。玩家扮演一位刚被大厂裁员的打工人，在不招募任何全职员工的前提下，通过咨询顾问、建立合作、经营人脉网络，将自己的 One Person Company 从零做到估值 $10M。

游戏的核心叙事装置是：**整个游戏界面就是主人公的那台 MacBook**。玩家不会"进入"一个游戏世界——他们只是在用这个人的电脑，操作他的 App，读他的邮件，和他认识的人发消息。

**AI 是游戏的核心驱动力**：每位 NPC 专家都有独立记忆，记住与玩家的所有历史对话；专家之间通过共享的「世界状态」互相感知玩家的行为，形成真实的人际网络效应。

### 1.2 核心体验目标

- **代入感**：玩家始终觉得自己就是那个人，而不是在操控一个角色
- **AI 驱动的真实感**：每位专家有独立人格和记忆，对话不可预测，每局体验独一无二
- **知识价值**：通过 Lenny 播客真实内容驱动的专家对话，让玩家在娱乐中获得真实的创业洞察
- **策略深度**：资源管理、时间分配、人脉网络三条线同时运转，每周决策都有实质影响
- **传播性**：游戏结局、专家关系图等天然适合截图分享

### 1.3 参考作品

| 作品 | 借鉴要素 |
|---|---|
| Her Story | 伪操作系统 UI、叙事沉浸感 |
| Reigns | 卡片化决策、资源平衡 |
| Disco Elysium | 对话深度、角色塑造 |
| 80 Days | 时间循环管理、路径多样性 |

## 二、核心玩法设计

### 2.1 游戏循环

游戏以「周」为基本单位循环推进：

```
每周开始
  └── 收到新邮件（事件牌）
  └── 分配本周 5 个行动点
  └── 执行行动（对话/外出/创作/休息）
  └── 周结算（资源变化 + 随机触发）
  └── 进入下一周
```

**游戏时长**：目标完整游玩约 60-90 分钟（约 20-30 个游戏内周期）

### 2.2 核心资源系统

| 资源 | 初始值 | 获得方式 | 消耗方式 | 归零后果 |
|---|---|---|---|---|
| 💰 现金 | ¥50,000 | 客户付款、合作分成 | 咨询费、外包费、生活成本 | 游戏结束 |
| ⚡ 精力 | 5点/周 | 每周自动重置 | 每个行动消耗1-3点 | 无法行动 |
| 🤝 人脉 | 5 | 社群互动、专家介绍 | 冷启动联系高质量专家 | 触达上限降低 |
| 🔥 信誉 | 10 | 交付质量、内容传播 | 违约、跳票 | 专家拒绝合作 |

**每周固定支出**：生活成本 ¥3,000（可通过「降低生活水准」选项减半，但精力上限-1）

### 2.3 行动系统

每周玩家可从以下行动类型中选择，总计不超过 5 个精力点：

**思考类（1点）**
- 收听播客 → 解锁专家信息 + 获得洞察碎片
- 复盘数据 → 信誉+1，下周某类行动成功率+10%
- 市场调研（Safari 搜索）→ 发现新专家 / 竞品情报

**社交类（1-2点）**
- 冷启动联系专家（1点）→ 成功率10-40%，受信誉影响
- 社群互动（1点）→ 人脉+2，偶发「隐藏专家搭话」
- 参加线下活动（2点）→ 人脉+5，但有「低质局」风险

**执行类（2-3点）**
- 开发产品原型（3点）→ 需技术支持或消耗现金外包
- 拜访潜在客户（2点）→ 受信誉影响的转化率
- 创作内容（1点）→ 小概率爆款；高质量内容可吸引专家主动联系

**恢复类（0点，消耗1周）**
- 休息 → 精力上限+1（临时），下周精力满状态开始

## 三、AI 驱动的专家系统

### 3.1 设计原则：AI 是叙事引擎，规则是游戏引擎

游戏进展的两个层面完全分离：

| 层面 | 驱动方式 | 说明 |
|---|---|---|
| **结构层**（数值/进展） | 预设规则 + 公式 | 挣多少钱、解锁什么、能不能成功——全部由确定性逻辑计算 |
| **叙事层**（对话/描述） | AI 实时生成 | 专家怎么说话、事件怎么措辞、结果怎么呈现——由 AI 赋予温度 |

这个分离确保了游戏节奏可控，同时每局体验独一无二。

### 3.2 NPC 记忆系统

每位专家维护独立的两层记忆结构，全部持久化在 localStorage：

```
NPC 记忆（每位专家独立）
├── 短期记忆：最近 5 条完整对话原文（保证上下文连贯）
└── 长期记忆：AI 自动压缩生成的「信念摘要」（随交互累积）
```

**长期记忆的生成机制**：每次对话结束后，AI 自动生成一句对该专家认知的更新，存为结构化条目：

```javascript
// NPC 记忆数据结构
{
  "lenny": {
    recentMessages: [           // 短期：最近 5 条原文
      { role: "user", content: "我的产品定价怎么定？" },
      { role: "assistant", content: "你的 B2B 客户..." }
    ],
    beliefs: [                  // 长期：AI 压缩的信念摘要
      "玩家专注 B2B SaaS 赛道，产品还未找到 PMF",
      "第 3 周资金紧张，倾向保守决策",
      "拒绝了一次 VC offer，重视独立性"
    ],
    relationship: "认识",       // 关系等级
    trustScore: 65,             // 信任分（0-100）
    lastInteraction: 6          // 最后互动的游戏周数
  }
}
```

### 3.3 NPC 间的信息流通

所有 NPC 共享一份**全局玩家行为日志**（playerProfile），这份日志会注入每次对话的 Prompt，实现「世界是连通的」效果：

```javascript
// 全局玩家行为日志（所有 NPC 可读）
const playerProfile = {
  direction: "B2B SaaS，目标用户是 PMer",
  actions: [
    { week: 2, type: "rejected_vc",       detail: "拒绝了红杉的 100 万投资" },
    { week: 4, type: "published_content", detail: "发表关于 PMF 的文章，获 200 转发" },
    { week: 5, type: "consulted",         expert: "lenny", topic: "定价策略" }
  ],
  reputation: 45,
  milestones: ["first_client"],
  communityContribution: 30
}
```

**效果示例**：玩家第 6 周联系 Andrew Chen，Andrew 的 Prompt 中包含「你听说这个人上周拒绝了 VC，还发了篇爆款文章」，于是 Andrew 会主动提及这件事，玩家会感到震惊——这个世界的人真的在互相聊你。

### 3.4 Prompt 构建规范

每次 NPC 对话，发给 AI 的完整 Prompt 按以下层次组装：

```
[SYSTEM — 角色定义]
你是 {专家名}，{职业背景}。
说话风格：{从播客提取的风格描述}。
你现在正在和一个刚被裁员、正在创业的人聊天。

[CONTEXT — 你对这个玩家的了解]
关于这个玩家，你已知道：
{npc.beliefs 列表}

[WORLD — 你从其他渠道听说的事]
你最近听说的关于他的事：
{playerProfile.actions 中与本 NPC 相关的条目}

[KNOWLEDGE — 你的专业知识（来自 Lenny 播客）]
关于玩家当前问题「{topic}」，你的核心观点是：
{匹配的知识碎片原文}
来源：{episode 信息}

[HISTORY — 本次对话上文]
{recentMessages 最近 5 条}

[玩家当前输入]
{playerMessage}

[CONSTRAINTS]
- 回复 3-5 句话，口语化
- 可以问一个追问
- 不要重复知识碎片的原文，用自己的语气转述
- 如果话题超出你的专长，直接说「这个你可以问问 {推荐专家}」
```

### 3.5 专家属性结构

```javascript
const EXPERT_TEMPLATE = {
  id: "lenny_rachitsky",
  name: "Lenny Rachitsky",
  title: "前 Airbnb PM，播客主持人",
  personality: "直接、好奇、数据驱动，喜欢追问「为什么」",
  expertise: ["PMF", "growth", "career", "B2C"],
  consultFee: 800,              // 每次咨询费用（¥）
  network: ["andrew_chen", "shreyas_doshi"],  // 可引荐的人脉
  knowledge: [                  // 来自播客的知识碎片
    {
      topic: "PMF",
      keywords: ["PMF", "留存", "product market fit"],
      fragment: "...(播客原文片段)...",
      source: "Lenny's Podcast, Ep.201",
      depth: 1                  // 1=基础，2=深度（需第二遍收听解锁）
    }
  ],
  unlockCondition: "initial"    // 初始可见
}
```

### 3.6 六种专家发现渠道

#### 渠道一：播客 App（主动发现，高信任起点）
- 收听 Lenny 节目（消耗1精力），收听过程触发「注意到某位嘉宾」提示
- 初始信任度 ⭐⭐⭐，专家类型：思想领袖型
- 特殊机制：同一集听第二遍解锁「深度洞察」，知识碎片 depth 升至 2

#### 渠道二：Safari 搜索（主动发现，低信任起点）
- 输入关键词触发伪搜索结果，搜索词影响专家池类型
- 初始信任度 ⭐，冷启动联系成功率 10-30%
- 特殊机制：搜索历史影响广告推送内容

#### 渠道三：社群 App（被动发现，中等信任）
- 类 Discord 界面，随机触发陌生人对话，需1轮验证对方价值
- 初始信任度 ⭐⭐，可挖到无公开影响力但实力强的「隐藏专家」
- 特殊机制：社群贡献值越高，主动搭话的人质量越好

#### 渠道四：广告推送（被动发现，消费者关系起点）
- 通知栏弹出付费课程广告，三种处理方式：购买 / 申请奖学金 / 绕过付费私信
- 初始信任度 ⭐⭐，需额外操作才能从「消费者」升级为「合作伙伴」

#### 渠道五：专家介绍专家（被动解锁，高信任跳跃）
- 合作关系≥2次后，专家主动在对话中提出引荐
- 初始信任度 ⭐⭐⭐⭐，联系成功率从默认值直升80%
- 特殊机制：形成「六度人脉」链，顶级隐藏专家需3-4跳引荐才能触达

#### 渠道六：内容吸引（被动触发，最高信任）
- 发布高质量内容后，专家主动发邮件联系，仅此渠道可触达
- 初始信任度 ⭐⭐⭐⭐⭐，内容质量由已积累的洞察碎片数量决定

### 3.7 三种合作方式

| 合作方式 | 成本 | 收益 | 解锁条件 |
|---|---|---|---|
| **咨询（单次）** | ¥500-2000现金 | 一次 AI 驱动的专属对话 | 信任度≥「认识」 |
| **合作（分成）** | 0现金，分出15-30%收益 | 专家带来渠道/背书 | 信任度≥「合作中」 |
| **雇用（长期）** | 每周固定费用 | 能力常驻+特技触发 | 信任度=「深度伙伴」 |

## 四、界面设计规范

### 4.1 整体框架：伪 macOS 桌面

游戏的完整 UI 是一个模拟 macOS 的桌面环境，包含：

- **菜单栏**：左侧显示当前游戏时间（第X周），右侧显示真实时钟、电量图标（随游戏时长减少）
- **桌面**：可放置文件/便签，壁纸随游戏进度变化
- **Dock 栏**：底部固定 App 图标，随游戏进度解锁新 App
- **窗口系统**：所有 App 以可拖拽窗口形式打开，支持多窗口叠加

### 4.2 App 与游戏机制对应表

| App 图标 | 游戏机制 | 视觉形态 | 解锁时机 |
|---|---|---|---|
| 📧 Mail | 事件系统 / 每周抉择 | 邮件客户端，三键选择 | 游戏初始 |
| 📅 Calendar | 行动点分配 | 周视图，拖拽行动卡 | 游戏初始 |
| 💬 Messages | 专家咨询对话（AI驱动） | iMessage 气泡界面 | 游戏初始 |
| 📊 Numbers | 资源面板 | 伪财务表格的数值看板 | 游戏初始 |
| 🎙️ Podcast | 发现专家 / 知识库 | 播客 App，收听解锁 | 游戏初始 |
| 🌐 Safari | 搜索专家 / 市场调研 | 伪浏览器，假搜索结果 | 第2周解锁 |
| 👥 Discord | 社群互动 | 频道列表+消息流 | 第3周解锁 |
| 🗂️ Contacts | 人脉网络图 | 关系图可视化 | 第一位专家认识后 |
| ⚙️ Settings | AI 模型配置 | 系统偏好设置风格 | 游戏初始 |
| 🔔 通知中心 | 随机事件插播 | 右上角滑入通知 | 全程 |
| 🖥️ Spotlight | 知识检索 | Cmd+Space 搜索框 | 全程 |

### 4.3 视觉规范

**调色板**
```
桌面壁纸：    #1A1A2E → #16213E（深蓝灰渐变，失业的沉重感）
窗口背景：    #FFFFFF / rgba(255,255,255,0.85)（毛玻璃效果）
主强调色：    #007AFF（macOS 蓝）
负面/危险：   #FF3B30
正面/成功：   #34C759
警告：        #FF9500
文字-主：     #1C1C1E
文字-次：     #8E8E93
文字-弱：     #C7C7CC
边框：        rgba(0,0,0,0.12)
```

**字体**
- 系统字体：-apple-system, BlinkMacSystemFont（零加载成本，还原 macOS 质感）
- 备用正文：Inter（Google Fonts CDN）
- 等宽字体：JetBrains Mono（终端/代码场景）

**动效原则**
- 窗口打开：scale(0.95→1) + opacity(0→1)，120ms ease-out
- 通知滑入：translateX(120%→0)，200ms cubic-bezier(0.16,1,0.3,1)
- AI 对话气泡：逐字打字机效果，18ms/字
- 所有状态变化带过渡，无硬切

### 4.4 叙事细节（不出戏设计）

- 菜单栏时钟显示**真实当前时间**
- 电池图标随游戏时长缓慢减少（纯装饰）
- 桌面壁纸变化节点：
  - 第1周：前东家公司 Logo 壁纸
  - 第一个付费客户：主角「自己换」成纯黑壁纸
  - 月收入 $5K：出现励志语录壁纸
  - 通关：用户自定义壁纸（象征真正的自由）
- 桌面右键菜单：可「整理桌面」触发隐藏成就
- Spotlight（Cmd+Space）：可检索游戏内任何知识内容

## 五、叙事系统

### 5.1 开场序列

1. 屏幕从黑暗中亮起，显示 macOS 登录界面
2. 玩家输入密码（引导设置，建立操作感知）
3. 桌面亮起，壁纸是前东家 Logo
4. 右上角弹出邮件通知：*"Your employment has been terminated effective today."*
5. 停顿3秒
6. 玩家点开邮件，读完全文（含 HR 官方措辞）
7. 桌面新增一个抖动图标：**「OPC 启动台」**
8. 点击图标，引导玩家完成 AI 配置（填入 API Key），游戏正式开始

### 5.2 事件牌系统（Mail App）

每周开始时收件箱出现1-3封邮件，分三类：

| 类型 | 说明 | AI 介入方式 |
|---|---|---|
| **A类：通知邮件** | 仅改变世界状态，无需选择 | AI 生成邮件措辞，每局不同 |
| **B类：抉择邮件** | 三个按钮，选择触发不同结果 | AI 生成发件人语气和细节 |
| **C类：机会邮件** | 本周不处理则过期消失 | AI 生成紧迫感措辞 |
| **特殊：引荐信** | 由专家介绍专家机制生成 | AI 以引荐专家的口吻撰写 |

### 5.3 游戏阶段与结局

**四个阶段**

| 阶段 | 触发条件 | 解锁内容 |
|---|---|---|
| 破冰期（第1-4周） | 初始 | 基础 App，前3位专家 |
| 起步期（第5-12周） | 第一个付费客户 | 合作模式，Safari，社群 |
| 成长期（第13-20周） | 月收入 $3K | 顾问市场，人脉图，PR 机会 |
| 扩张期（第21-30周） | 月收入 $10K | 融资选项，收购邀约，出口机会 |

**五种结局**
1. **OPC 之王**：估值 $10M，保持单人 → 通关正结局
2. **转型建队**：接受 VC，建立真正的团队 → 支线结局
3. **并购退出**：被大厂收购 → 讽刺性结局
4. **躺平悟道**：主动降低生活标准，找到内心平静 → 隐藏结局
5. **资金耗尽**：Game Over → 重新开始，保留部分人脉记忆

## 六、Lenny 数据集成方案

### 6.1 数据处理流程

```
原始转录文本（.md）
  └── 提取「问题-洞察」对
  └── 按主题分类（PMF / 增长 / 融资 / 产品 / 职业）
  └── 按嘉宾归类，提取说话风格特征
  └── 生成 knowledge_base.json（内嵌进游戏 HTML）
  └── 在 Prompt 中作为 KNOWLEDGE 层注入 AI
```

### 6.2 知识碎片数据结构

```javascript
{
  topic: "PMF",
  keywords: ["PMF", "留存", "product market fit", "retention"],
  fragment: "...(播客原文片段，100-200字)...",
  source: "Lenny's Podcast, Ep.201",
  expert: "lenny_rachitsky",
  depth: 1    // 1=基础对话可用，2=深度收听后解锁
}
```

### 6.3 知识价值设计

- 每次 AI 回复末尾标注真实播客来源，增强可信度与教育价值
- 「洞察碎片」可在 TextEdit（笔记 App）中查看累积记录
- 游戏结束后可导出「我的创业笔记」—— 真实知识摘要，游戏之外可继续使用

```
├── CSS     所有样式与动画

└── Data    专家库 / 事件库 / 知识碎片（内嵌 JSON）

运行时依赖：
├── 用户的 AI API Key（必须）
├── 现代浏览器（Chrome 90+ / Safari 14+ / Firefox 88+）
└── 无需 Node.js / 无需服务器 / 无需数据库
```

### 7.2 AI SDK 支持

使用 **Vercel AI SDK**（浏览器兼容版），支持用户自选 AI 提供商：

```javascript
const AI_PROVIDERS = {
  claude:  {
    baseURL: "https://api.anthropic.com",
    model: "claude-3-5-haiku-20241022",   // 默认：快速便宜
    headers: { "anthropic-dangerous-direct-browser-access": "true" }
  },
  openai:  { baseURL: "https://api.openai.com",    model: "gpt-4o-mini" },
  gemini:  { baseURL: "https://generativelanguage.googleapis.com", model: "gemini-2.0-flash" }
}
```

玩家在 Settings App 中配置，API Key 仅存于 localStorage，不经过任何第三方服务器。

**降级策略**：未配置 API Key 时，系统使用预设模板回复，保证游戏可玩（体验降级但不崩溃）。

### 7.3 数据存储方案

| 数据类型 | 存储位置 | 大小估算 | 说明 |
|---|---|---|---|
| 专家静态数据 | HTML 内嵌 JSON | ~50KB | 只读，不变 |
| 事件文本库 | HTML 内嵌 JSON | ~100KB | 只读，不变 |
| Lenny 知识碎片 | HTML 内嵌 JSON | ~200KB | 只读，不变 |
| 玩家游戏状态 | localStorage | ~10KB | 动态，持久化 |
| NPC 记忆（全部） | localStorage | ~20KB | 动态，持久化 |
| 玩家行为日志 | localStorage | ~5KB | 动态，持久化 |
| AI 配置（Key等） | localStorage | <1KB | 敏感，仅本地 |

**总计**：单文件约 400KB，localStorage 占用约 35KB，浏览器侧零感知。

## 九、非功能性需求

### 9.1 性能要求
- 首屏加载 < 2秒（单 HTML 文件，无外部资源依赖）
- 动画帧率稳定 ≥ 60fps
- AI 对话响应启动 < 500ms（流式输出，第一个字符即开始显示）
- 游戏状态本地操作响应 < 50ms

### 9.2 兼容性
- 桌面浏览器：Chrome 90+、Safari 14+、Firefox 88+
- 移动端：自适应布局，功能完整，交互简化
- 无障碍：键盘可导航，WCAG AA 对比度

### 9.3 隐私与安全
- 所有游戏数据本地运行，无服务端
- API Key 仅存于 localStorage，永不上传，游戏代码可审计
- 不收集任何用户数据，无埋点，无统计

### 9.4 离线能力
- 无 API Key 时游戏仍可运行（降级为预设模板对话）
- 所有静态数据内嵌，无需网络即可启动
